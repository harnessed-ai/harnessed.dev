# Website

Website for tools and plugins — a landing page plus dedicated pages for two
tmux plugins (`tmux-ai-sessions-restore` and `tmux-nudge`), styled with the
**Nocturne** design system (dark, left-aligned, code-forward).

The pages are plain, hand-editable HTML — no build step, no framework, no
bundling. Open a file and edit it.

## Pages

| File | Page |
| --- | --- |
| `index.html` | Home / landing |
| `sessions-restore.html` | tmux-ai-sessions-restore |
| `nudge.html` | tmux-nudge |

Each page is a standalone HTML file that pulls in the shared theme with a single
`<link rel="stylesheet" href="styles.css">`. They cross-link to each other via
the top nav.

> **History:** these pages were originally generated by Claude Design as
> self-contained `.dc.html` bundles (HTML + React + a JS runtime gzip-embedded
> in each file). They've been de-bundled into the plain HTML you see here so
> that humans can edit them too. The old bundle format is gone.

## Theme

- `styles.css` — the Nocturne theme. Holds all the design tokens
  (`--color-*`, `--font-*`, `--space-*`, `--radius-*`, `--shadow-*`) plus the
  base type and component classes (`.nav`, `.card`, `.btn`, `.hr`, …). This is
  the single source of truth for the look; every page reads from it.

**Keep `styles.css` next to the pages.** If it goes missing, the CSS tokens
fall back to browser defaults and the pages render unstyled (white background,
serif text). Inter is pulled from Google Fonts over the network (via an
`@import` at the top of `styles.css`), so the exact typeface needs a connection;
colors and layout work offline.

## Media assets

Some pages reference media that lives beside the HTML files:

- `demo.gif` — demo shown on `sessions-restore.html`.
- `nudge-demo.gif` — demo shown on `nudge.html`.

These are loaded by relative path, so they must stay next to the pages. If you
move or rename a page, keep its assets with it.

## Interactivity

The two plugin pages have a "Copy" button that copies the install snippet to the
clipboard. It's a small inline `copyInstall()` function at the bottom of each of
those files (plain vanilla JS, no dependencies) — edit it there if the snippet
changes.

## Editing a page

It's just HTML. Open `index.html` / `sessions-restore.html` / `nudge.html` in
your editor and change the markup directly. Most styling comes from classes and
CSS variables defined in `styles.css`; prefer those over hard-coded hex/px/font
values the tokens already carry. Per-page one-off CSS (e.g. keyframes) lives in a
`<style>` block in that page's `<head>`.

The shared nav and footer are currently duplicated across the three pages — if
you edit one, update the others to match.

## Viewing the pages

### Quick look (open a file directly)

Double-click any page, or from a terminal:

```sh
open index.html      # macOS
```

`file://` works because there's no build step — relative paths resolve to
`styles.css` and the GIFs in the same folder.

### Local web server (recommended)

A local server mirrors how the site is hosted:

```sh
cd /path/to/website
python3 -m http.server 8000
# then open http://localhost:8000/index.html
```

Stop it with `Ctrl-C` (or `lsof -ti:8000 | xargs kill`).

## Updating the theme

The theme is derived from the Nocturne design system. To restyle, edit the
tokens at the top of `styles.css` — every page reads from them, so no page
markup needs to change. Don't hard-code hex/px/font values the tokens already
carry.
